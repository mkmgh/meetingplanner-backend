Events to listen

verifyUser event
This event ("verify-user") has to be listened on the user's end to verify user authentication.
user will only be set as online user after verification of authentication token.
auth-error event
This event ("auth-error") has to be listened on the user's end if in case authentication error.
onlineUsers event
This event ("onlineUsers") has to be listened on the user's end to identify all available users that are currently online.
The output will be an object, object has key as userId and value as full name.
userId event
This event ("userId") has to be listened to get the updates from Admin.

Events to emit

set-user event
This event ("set-user") has to be emitted when a user comes online.
User can only be set as online only after verification of authentication token. Which you have pass here. The following data has to be emitted:
authentication token
disconnect event
This event ("disconnect") has to be emit after user logout from the application and userlist will refresh again.
notifyUpdates event
This event ("notifyUpdates") has to be emit to notify general users after admin does any updation, creation or deletion on meeting
